# Fix: clients-screen.js ID mismatch causing edit client crash

## Problem
When clicking the edit button on a client card, the error occurs:
```
clients-screen.js:194 Uncaught TypeError: Cannot set properties of null (setting 'value')
```

## Root Cause
ID mismatch between HTML and JavaScript:
- HTML (line 639): `<textarea id="client-notas">`
- JS (line 194): `document.getElementById("client-notes").value = ...`

The `editClient()` function tries to set value on an element that doesn't exist.

## Fix
Change line 194 in `inventory/js/screens/clients-screen.js` from `"client-notes"` to `"client-notas"` to match the HTML element ID.

## Verification
- The code at line 214 already uses `"client-notas"` correctly when reading the value
- This is a simple one-character fix: `notes` → `notas`
