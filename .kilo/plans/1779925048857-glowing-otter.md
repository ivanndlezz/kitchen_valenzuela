# Plan de Implementación: Inventario Persistente Creado por Escaneo

## Contexto

Se requiere una **interfaz independiente** para gestión de inventario autónolo (sin JSON estático). Los productos se origina escaneando códigos de barras y persisten en `localStorage`. El drawer soporta dos vistas: Ficha Técnica y Formulario de Edición mediante glider tabs.

## Módulo Independiente a Crear

Se crea `inventory/` como un sistema autónomo, reutilizando recursos compartidos pero sin modificar `home/`.

---

### Paso 1 — Estructura de Directorios

```
inventory/
├── index.html    # UI shell completa
├── inventory.css # CSS propio (tokens heredados)
└── inventory.js  # Lógica principal
```

### Paso 2 — Interfaz HTML (`inventory/index.html`)

1. Header con logo, título y botón **Escanear Código** (único entry point).
2. Dashboard de métricas: Equipos Registrados, Unidades Totales, Alertas, Marcas.
3. Filtros: búsqueda, pestañas de categoría, select de marca, filtro de stock, ordenamiento, toggle vista.
4. Grid/Lista de productos con tarjetas.
5. **Scanner Modal** con:
   - Scrim y overlay premium con `backdrop-filter: blur(10px)`.
   - Tabs: **Foto (System)** y **Video (Apple)**.
   - Input file con `capture="environment"` y botón Tomar Foto.
   - Lector de cámara en vivo (`#reader`) y botón Iniciar Cámara.
6. **Detail Drawer**:
   - `.drawer__tabs` con pestañas **Producto** y **Editar**.
   - `.drawer__view-product` (ficha técnica con barcode SVG).
   - `.drawer__view-form` (formulario de edición).
7. Empty State premium con ilustración de código de barras y CTA "Escanear Primer Artículo".

### Paso 3 — Estilos CSS (`inventory/inventory.css`)

Reutilizar tokens BEM de `app.css` y definir:

1. **Scanner Modal**: `.scanner-modal`, `.scanner-modal--active`, `.scanner__tabs`, `.scanner__reader-box`.
2. **Glider Tabs**: `.drawer__tabs`, `.drawer__tab--active`, transiciones.
3. **Formulario**: `.drawer__form`, inputs con `:focus` glow, botones de acción.
4. **Empty State**: ilustración SVG barcode, botón CTA prominente.

### Paso 4 — Lógica JavaScript (`inventory/inventory.js`)

#### 4.1 Estructura de Producto
```js
{
  id: String, codigo: String, nombre: String, marca: String,
  categoriaCodigo: "01|02|03|other", unitCode: String,
  costo: Number, precio: Number, alertaCantidad: Number, stock: Number,
  barcodeType: "code128", descripcion: String, especificaciones: String,
  especial3: String, especial4: String, especial5: String, especial6: String
}
```

#### 4.2 Persistencia
- `saveProductsToStorage()` → `localStorage.setItem("kv-catalog-products", JSON)`.
- `loadProductsFromStorage()` → inicializa desde localStorage o `[]`.

#### 4.3 Motor de Escaneo
Adaptar clases de `scanner_foto_pro.html`:
- `TabController` (tabs Foto/Video).
- `Html5QrcodeStrategy` + `BarcodeDetectorStrategy` (foto).
- `VideoScanController` (cámara en vivo).
- Flujo: detectar código → buscar en productos → crear/actualizar → abrir drawer.

#### 4.4 Glider Tabs
- Alternar `.drawer__tab--active`.
- Mostrar `.drawer__view-product` / `.drawer__view-form`.

#### 4.5 Formulario de Edición
- Generar form dinámico con campos: Nombre, Marca, Categoría, Unidad, Costo, Precio, Alerta, Stock, Descripción, Especificaciones, Ubicación.
- Al guardar: actualizar producto, persistir, recalcular métricas, volver a vista Producto.

### Paso 5 — Verificación Manual

1. Carga inicial vacía: dashboard 0s, empty state con CTA.
2. Escanear foto/video: producto creado, drawer abierto automáticamente.
3. Editar: cambiar pestaña, modificar, guardar → catalogo/métricas reactivas.
4. Refresh: datos conservados en localStorage.
